# MVP範囲と開発計画

## 1. 今回の縦切り試作

相場システム全体を作る前に、最も危険な仮説である
「情報を相場判断へ変えること自体が面白いか」を検証する。

### 含めるもの

- 30日間の短いキャンペーン
- 1種類の米
- 天候、入荷、需要による価格変動
- 精度と公開時期が異なる人物情報
- 現物米の売買
- 資金、在庫、信用、町の暮らし
- 状態に応じた複数の結末
- 値動きの理由を振り返る相場日誌

### 今回は含めないもの

- 先物、限月、証拠金
- 複数産地と品質
- 注文板の詳細シミュレーション
- 店舗拡張
- 人間関係イベント
- セーブ
- 本番用グラフィックと音響

これらは、基本ループに手応えがあると確認してから追加する。

## 2. 開発マイルストーン

### M0: 企画検証

- 紙上の価格モデル
- 情報、売買、翌日への進行
- 15分で完走可能
- 5人以上の観察テスト

### M1: 市場の手触り

- 簡易注文板
- 出来高と流動性
- 複数の米
- 現物の保管費用と劣化
- 相場日誌の改善

### M2: 帳合米

- 3つの限月
- 証拠金、値洗い、追証
- 現物との価格差
- ヘッジと裁定
- 初心者向け導線

### M3: 生活RPG

- 町の移動
- 人物との関係
- 店と蔵の拡張
- 章立てされた物語
- 季節と長期需給

### M4: 公開体験版

- 本番アートの一部
- 約60分のシナリオ
- チュートリアル
- 設定、セーブ、アクセシビリティ
- Steamページ用素材

## 3. プレイテストで記録すること

- 最初の売買までの時間
- 各日に確認した情報数
- 価格予想と実際の方向
- 売買量と資産推移
- 町の暮らしを意識した選択
- 混乱した言葉や画面
- 終了後に説明できた因果関係

## 4. 次の実装優先順位

1. 価格モデルが納得できること
2. 情報ごとに判断価値が違うこと
3. 売買操作が速く、結果が明瞭なこと
4. 町の暮らしが数字以上に感じられること
5. 先物を追加しても理解可能な土台を作ること
