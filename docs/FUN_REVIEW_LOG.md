# 面白さレビュー記録

## 2026-07-06 — M0初回レビュー

### 対象

情報収集、現物売買、一日の進行。

### 良い点

- 人物の噂を価格予測へ使う、作品固有の基本体験が成立している
- 利益、店の信用、町の暮らしが別の評価軸になっている
- 翌日の値動きと原因を相場日誌で確認できる

### 問題1: 調査と取引の駆け引きがない

| 項目 | 内容 |
|---|---|
| 観察 | 情報収集は一刻を使うが、売買には刻を使わない |
| 原因仮説 | 取引回数が無制限で、調査と行動を選ぶリスク・リターンが存在しない |
| 改善 | 仕入れ・売却も一刻を消費する。一日三刻を全行動で共有する |
| 成功条件 | 「もう一件聞くか、今の情報で取引するか」で迷う |
| 判定 | 実装。プレイテスト待ち |

### 問題2: 噂の日付の意味が伝わらない

| 項目 | 内容 |
|---|---|
| 観察 | 「〇日目の話」の数字に意味があるか、ユーザーから質問があった |
| 原因仮説 | システム内部の日数を表示しており、何の予測かが見出しにない |
| 改善 | 「明日の入荷見通し」「明後日の需要見通し」のように、相対日と対象を表示する |
| 成功条件 | 説明を読まず、いつ・何が動く噂か判断できる |
| 判定 | 実装。プレイテスト待ち |

### 次回確認

- 三刻のうち、情報収集と売買へ何刻ずつ使われるか
- 情報ゼロの投機と、情報を三件集めて取引しない行動の割合
- 「一刻」の消費に気づけるか
- 噂が示す日と、値動きの発生日を対応づけられるか

## 2026-07-06 — 一画面完結UI

### 対象

デスクトップ版の画面構成、噂と相場日誌の表示量。

| 項目 | 内容 |
|---|---|
| 観察 | 相場、情報、日誌、店じまいを確認するために縦スクロールが必要だった |
| 原因仮説 | 大きなヘッダー、固定高グラフ、履歴の全件表示が、判断に必要な操作を画面外へ押し出していた |
| 改善 | デスクトップでは画面高へレイアウトを適応させ、全操作を一画面へ収める。噂は最新3件、日誌は最新4件を表示する |
| 成功条件 | スクロールせず、情報収集、売買、値動き確認、翌日進行を繰り返せる |
| 判定 | 実装。公開版での確認待ち |

### 設計判断

- 文字を一律に小さくせず、余白と固定高を優先して圧縮する
- 過去ログより、現在の判断材料を常時表示する
- 幅980px以下では可読性を優先し、既存の縦長レスポンシブ表示を維持する

## 2026-07-06 - 帳合米・三日物

### 対象

堂島米市場の投機、流動性、差金清算を体験へ変換する。

| 項目 | 内容 |
|---|---|
| 観察 | 噂が2～3日先を示しても、現物取引では保管する米と資金がなければ予想を活かしにくい |
| 原因仮説 | 現物の物流と価格予想が同じ操作にまとめられ、堂島市場固有の「価格そのものを取引する」体験がない |
| 改善 | 現物不要、一口10俵、証拠金25%、三日後差金清算の帳合米を追加する |
| 成功条件 | プレイヤーが2～3日先の噂を根拠に買建て・売建てを選び、現物との違いを説明できる |
| 判定 | 実装。プレイテスト待ち |

### リスクとリターン

- 少ない証拠金で大きい値動きを受け取れる
- 現物在庫が不要な代わりに、満期まで撤回できない
- 情報を集めるほど精度は上がるが、建てるための一刻が必要
- 投機は直接町へ米を出さないため、現物放出による信用上昇は得られない

## 2026-07-06 - 火縄大引と立用

### 対象

一日の締め、三刻の配分、堂島らしい手触り。

| 項目 | 内容 |
|---|---|
| 観察 | 店じまいは結果を進めるだけのボタンで、毎日の判断が終わった後に緊張がなかった |
| 原因仮説 | 堂島固有の価格形成が計算処理の裏側に隠れ、情報・売買・翌日の結果が儀式としてつながっていない |
| 改善 | 8秒の火縄大引を追加。注文が片側へ偏った日は、一刻を使う反対注文か、立用による終値据え置きかを選ぶ |
| 成功条件 | プレイヤーが「大引のために一刻を残すか」を日中から考え、建玉と翌日の噂を根拠に立用の是非を迷う |
| 判定 | 実装。プレイテスト待ち |

### 面白さの構造

- 調査を増やすほど予測は正確になるが、大引へ参加する一刻がなくなる
- 相場を成立させれば価格が動き、会所での信用も上がる
- 自分に不利な値動きなら、あえて流動性を供給せず立用を選べる
- 火縄の視覚的な残り時間が、抽象的な終値決定を毎日の見せ場にする

## 2026-07-07 - Chrome実表示向け一画面UIと本日の商い

### 対象

実Chromeの低い表示領域での一画面完結、火縄大引後の振り返り表示。

| 項目 | 内容 |
|---|---|
| 観察 | ユーザーのChrome画面では、取引数量以降が下に切れ、仕入れ・売却・帳合米が見えにくかった。プレイ検証では、火縄大引後に「噂が当たったか」「今日の判断で何が増減したか」が日誌だけでは流れやすかった |
| 原因仮説 | ブラウザ上部UIやタスクバー込みの実表示領域に対して、公開版のCSSキャッシュと縦方向の余白が大きかった。結果表示が日誌文に埋もれ、プレイヤーの判断と結果の結びつきが弱かった |
| 改善 | CSS/JSにキャッシュ避けを付け、PCレイアウトを全体的に圧縮。さらに高さ760px以下の専用圧縮ルールを追加し、相場日誌内に「本日の商い」カードを追加。火縄大引後に相場結果、噂の的中数、総資産・手元金・蔵の増減、信用・町の変化、帳合米清算を表示する |
| 成功条件 | 1280x650の表示領域で、町を歩く、仕入れ、売却、帳合米、火縄大引がスクロールなしに見える。火縄大引後、プレイヤーが自分の判断の成否を日誌本文を読まずに説明できる |
| 判定 | 実装。ローカル1280x650と公開Chrome 1028x513相当で確認済み |

### 桜井レビュー

- 体験の核: 「噂を読む→張る→火縄で結果を見る」の後に、結果カードで判断の成否を返すため、相場を読んだ体験が数字だけで流れにくくなった。
- リスクとリターン: 帳合米を建てると手元金が減り、総資産や清算結果に反映されるため、張ったリスクが見えやすくなった。
- 分かりやすさ: 噂の的中数を出すことで、誰の話が当たったかを後から学習しやすい。ただし低い画面では表示行数を絞るため、詳細は日誌との併用が必要。
- 手応えとフィードバック: 火縄大引の直後に、相場・噂・資産・信用・町の変化がまとまって返る。操作結果の反射が強くなった。
- テンポ: モーダルではなく常設カードにしたため、確認クリックを増やさずテンポを保てる。
- ゲーム性以外の面白さ: 番頭の記録に「本日の商い」が増え、帳簿をめくるごっこ遊びが少し強まった。

### 次回確認

- 総資産と手元金の差、とくに帳合米の証拠金拘束が初見で誤解されないか
- 低い画面で隠した詳細行が、面白さの理解を損ねていないか
- ユーザー自身のChromeで通常更新したとき、古いCSSキャッシュが解消されるか

## 2026-07-09 - 帳合米の総資産導線

### 対象

帳合米の証拠金拘束、評価損益、満期までの見通しを、取引中に迷わず読めるようにする。

| 項目 | 内容 |
|---|---|
| 観察 | 1280x650 のローカル実機表示で、1日目に情報収集→現物仕入れ→火縄大引、2日目に帳合米の買建てを試した。帳合米を建てると手元金は 1,100文→862文へ減る一方、総資産は 1,955文のまま維持され、証拠金 238文預かり・評価 0文・満期 5日目/あと3日が同じ箱で読めた。縦スクロールは発生せず、主要操作は一画面で完結した。 |
| 原因仮説 | これまでは手元金の減少だけが強く見え、証拠金が消えたのか、資産として残っているのかをプレイヤーが自力で補完する必要があった。現物と帳合米の違いが説明文にはあっても、取引直後の画面変化へ落ちていなかった。 |
| 改善 | 状態帯に総資産を常設し、帳合米欄に証拠金・評価損益・満期を常時表示する。建玉の有無に応じて、手元金と総資産の見方を一行で案内する。ヘルプ文にも同じ因果を追加する。 |
| 成功条件 | 初見プレイヤーが「手元金は減ったが、証拠金は総資産に残っている」と画面だけで説明できる。現物売買と帳合米の違いを、ヘルプを閉じた後でも見失わない。 |
| 判定 | 維持。公開版で初見確認待ち。 |

### 桜井レビュー

- 体験の核: 噂を読んだあと、現物で蔵を動かすか、帳合米で価格だけを張るかの違いが、総資産と帳合米欄の差として即座に見える。読みをどの賭け方へ変えるかが前より考えやすい。
- リスクとリターン: 帳合米を建てた瞬間に手元金だけが減り、総資産は維持されるため、拘束資金と評価損益の構造を誤読しにくくなった。少額資金で張れる利点と、満期まで拘束される不自由さが同時に伝わる。
- 分かりやすさ: 証拠金・評価・満期を一箇所へ束ねたことで、何が減って、何が残り、いつ返るかが追いやすい。専門用語は残るが、数字の意味が判断と結びついた。
- 手応えとフィードバック: 帳合米の建玉直後に、手元金、総資産、帳合米欄、残り刻が同時更新される。操作の結果がワンテンポで返るため、説明を読まなくても変化を掴める。
- テンポ: 新しいモーダルや確認操作を増やさず、既存の取引箱だけで理解補助を入れられた。テンポの悪化は感じなかった。
- ゲーム性以外の面白さ: 番頭が帳面に預け金と満期を書き付けている感触が少し増し、商いのごっこ遊びとしても納得感が上がった。

### 次回確認

- 初見プレイヤーが「総資産」を利益そのものと誤解せず、現物在庫込みの値として理解できるか
- 評価損益がマイナスになった局面で、帳合米欄の文字だけで危険が十分伝わるか
- 公開版でも同じ一画面導線がキャッシュ残りなしで表示されるか
