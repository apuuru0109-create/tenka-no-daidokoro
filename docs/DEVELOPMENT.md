# 開発・公開手順

## 通常の作業

1. ゲームを変更する
2. 変更箇所を含む主要操作を、実際にプレイして確認する
3. `FUN_REVIEW.md`の全観点で**桜井レビュー**を行う
4. 観察・原因仮説・改善・成功条件・判定を`FUN_REVIEW_LOG.md`へ記録する
5. 簡易テストを実行する
6. `main`ブランチへ反映する
7. GitHub PagesがWeb版を自動更新する
8. 公開版を実際に遊び、変更箇所を確認する

ゲームへ実装を加えた回は、桜井レビューの実施と記録を省略しない。
レビュー記録がない変更はコミット、push、公開を行わない。

## ローカルテスト

```powershell
node tests/smoke.mjs
```

## 公開対象

ゲーム画面が読み込む公開用ファイルは次の3点。

- `index.html`
- `game.js`
- `styles.css`

GitHub Pagesは`main`ブランチのルートを配信元とする。

## 公開の条件

- スモークテストが成功している
- 実際の操作確認に基づく桜井レビューが完了している
- 桜井レビューの結果が`FUN_REVIEW_LOG.md`に記録されている
- 追加機能の目的と成功条件がレビュー記録に残っている
- デスクトップ版の主要操作がスクロールなしで完結する
- 公開後に主要操作を一周確認している
