# 開発・公開手順

## 通常の作業

1. ゲームを変更する
2. 面白さレビューを行い、`FUN_REVIEW_LOG.md`へ記録する
3. 簡易テストを実行する
4. `main`ブランチへ反映する
5. GitHub Actionsがテスト後にWeb版を自動更新する
6. 公開版を実際に遊び、変更箇所を確認する

## ローカルテスト

```powershell
node tests/smoke.mjs
```

## 公開対象

GitHub Pagesへ配置するのは次のファイルだけ。

- `index.html`
- `game.js`
- `styles.css`

設計資料、テスト、リポジトリ設定はWebサイトの配信物には含めない。

## 公開の条件

- スモークテストが成功している
- `FUN_REVIEW.md`の観点でレビュー済み
- 追加機能の目的と成功条件がレビュー記録に残っている
- 公開後に主要操作を一周確認している
